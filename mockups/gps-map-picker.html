MAP PICKER MODAL · OPENS AFTER "PICK ON MAP"

HOW IT WORKS

1. Click anywhere on the map → pin drops, lat/lng updates, reverse-geocode pulls the place name.

2. Type in the search bar → live results from OpenStreetMap's free Nominatim service. Click a result → map pans + pin moves.

3. Right rail: Recently used (last 8) + Saved places (you can name + save the current pin).

4. Bottom-left readout shows the resolved place name + exact coords. USE LOCATION commits + closes.

WHAT THE PIN ACTUALLY DROPS

When the user clicks USE LOCATION, we set:

iphoneGpsState = {
  lat: 30.044420,
  lng: 31.235712,
  label: "Cairo, Egypt",  // reverse-geocoded for the sidebar status
  randomize: true,  // ±5km jitter on each export per existing behavior
}

EXIF write path doesn't change — same getGpsForExport() already converts these to GPSLatitude/GPSLongitude/GPSLatitudeRef/GPSLongitudeRef tags. The map picker only changes the UX of setting the lat/lng, not what gets written.

STACK CHOICE

OPENSTREETMAP + LEAFLET (RECOMMENDED)

Why: tiles + geocoding are free, zero API keys, MIT-compatible. tile.openstreetmap.org serves PNG tiles directly; nominatim.openstreetmap.org does forward + reverse geocoding (1 req/sec rate limit, fine for one-user-at-a-time desktop app).

Render: Leaflet.js (~40KB gzipped, MIT license) — drop-in interactive map. No build-step changes; just a script tag. The hidden Tauri webview (or main webview if we put the modal here) loads the tiles.

Caveat: we should send a polite User-Agent header per OSM's usage policy and respect rate limits. Easy in the fetch call.

Alternatives: Mapbox (free up to 50k loads/mo, then $$); Google Maps (paid); Apple MapKit JS (free with Apple Developer account, but JS API requires online verification). OSM/Leaflet is the only fully-free no-account option.
