Things that don't add new tabs but make the existing app feel like a daily tool. Sorted by area, tagged by effort and impact. Pick the ones that resonate.

WORKFLOW & SHORTCUTS

01 Keyboard shortcuts everywhere EFFORT SIMPACT HIGH

Bind common actions: Cmd+E export, Cmd+, settings, Cmd+T caption template menu, Cmd+Shift+R trial reel randomizer, Cmd+1/2/3/4 switch tabs, Esc close any modal, ? show all shortcuts. Adds a small ? in the header that pops a cheat sheet.

02 Undo / Redo across edits EFFORT MIMPACT HIGH

Cmd+Z to undo the last 50 actions: text edit, slider change, filter apply, font swap. The single most-asked feature for any creative tool — currently you have to reset entire sections.

↶ Undid: applied "Vivid" filter 42 / 50 in history
⌘Z · ⌘⇧Z
03 Auto-save drafts (and recovery) EFFORT MIMPACT HIGH

Snapshot the editor + carousel state to disk every 30s. On launch: "Recover unsaved session?" if the app was killed mid-edit. Removes the panic when the app crashes mid-export or you close the wrong window.

RECOVER

04 Recent files / Recent exports menu EFFORT SIMPACT MED

A Recents dropdown next to upload. Last 10 sources you opened + last 10 things you exported. Reopen with one click — no more "where did I save that".

Last 8 colors used appear as swatches under the color picker. Cuts the constant re-picking when you have a brand palette you reuse.

Every icon-only button gets a hover label after ~600ms. Removes the "what does this button do?" tax for new users + power users who haven't memorized everything.

Checkbox row in the export dialog: 4:5 (IG feed), 9:16 (story/reel), 1:1 (carousel), 1.91:1 (link card). Tick the ones you want, get all of them in one export pass. Creator workflow saver.

09 Export-complete notification + reveal/open buttons EFFORT SIMPACT MED

Desktop notification when a long export finishes (videos can take a minute+). The toast offers SHOW IN FOLDER + OPEN. Means you can alt-tab to reply to a message and not miss when it's done.

STATE & CRASH SAFETY

11 Window state remembered EFFORT SIMPACT MED

App opens at the same window size and position as last time. Currently it always opens at default (1400×900). Annoying on multi-monitor setups.

window.y = 180
window.width = 1840
window.height = 1080
window.maximized = false

12 "Reveal in folder" everywhere EFFORT SIMPACT MED

Right-click any source image in the editor / carousel → Reveal in Finder/Explorer. Same backend as the saved-export modal already uses (the reveal_in_folder Rust command). Two-line addition.

13 Better empty states EFFORT SIMPACT MED

Each empty zone (carousel before any images, inspector before any file, trending before any creators) gets a guided empty state: icon + what this is + 1-2 sample actions. New users + returning-after-a-week users get oriented faster.

14 First-launch onboarding tour EFFORT MIMPACT MED

5-step popover tour the first time the app opens: drag a file, set caption, apply filter, hit export, here's the cogwheel. Skippable + dismissible permanently. Most apps that don't have one are leaving 30%+ of new users stuck on screen 1.

STEP 2 OF 5
Type your caption here. The text live-renders on the preview as you type — drag it on the canvas to reposition.

15 Better error messages (not stack traces) EFFORT SIMPACT MED

Audit every error path: replace tech-y messages like "FFMPEG ENCODE FAILED: Conversion failed" with what happened + what to do. "Couldn't encode the video — your source file may be corrupted. Try opening it in QuickTime to verify."

EXPORT FAILED
The video encoder couldn't read frame 142 — usually means the source file is partially corrupted.
Try: Open the source in QuickTime to verify it plays end-to-end.
SHOW LOG RETRY

18 Apple Developer signing + notarization EFFORT MIMPACT HIGH

Already on the todo list — pulling forward. Eliminates the xattr -cr dance for every fresh download. $99/yr Apple Developer account, certs in GitHub secrets, ~6 lines added to the workflow. Tauri-action handles the rest. Whoever installs your app shouldn't need to touch Terminal.

Currently:
$ xattr -cr "/Applications/Snap Caption Studio.app"
After signing:
✓ Double-click → app opens. Done.

If the app crashes, write a small report file with last 200 log lines + system info. Next launch: "App crashed last session — view report?" Lets you debug remote bug reports without asking users to find log files manually.
